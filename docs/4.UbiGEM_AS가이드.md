1. 유비젬에 사전 지식
    1> 유비젬 키락
        1.1> usb 타입 키락 1.0(신형 : 시리얼번호 길이 정상)
            - 이 키락은 지금은 배포하지 않는 1.0 을 사용하는 업체에서 사용하는 키락이다.
            - usb 타입으로 탐색기에서 usb드라이버 로 잡힌다.

        1.2> usb 형태 동글 키락 2.0
            - 신규 업체는 모두 2.0 으로 배포한다.
            - 모양만 usb 이고, 실제 usb 가 아니다.

    2> 유비젬 키락 없을 때 3시간 마다 체크
        2.1> 3시간에 한번씩 키락을 체크해서 연결을 강제 해제 한다.
            - 강제로 해제하면서 EXCP(예외) 에러를 발생 시킨다.
            - 고객의 로그에서 EXCP 가 있어도, 키락 문제일 경우가 많다.

    3> 신규업체 유비젬 배포버전
        3.1> UbiGEM_20221221.zip : 이 버전이 최종버전 이다.

    4> 유비젬 키락 2.0 vs 1.0 구분 법
        4.1> C:\Program Files\UbiSam\UbiGEM\BIN : 유비젬 설치 폴더
            - "UbiSam.Net.KeyLock.Checker.exe" 이 존배하면 유비젬 2.0
            - 없으면 유비젬 1.0 으로 판단한다.
            - 단, 유비젬 1.0 은 신규 업체에는 더이상 배포하지 않는다.

        4.2> "UbiSam.Net.KeyLock.Checker.exe" 의 실행을 권장한다.
            - 유비젬 2.0 키락을 usb 에 꼽고, 
                이 프로그램을 실행해서, 키락이 정상적인지 확인 하도록 고객에게 알렺준다.

    5> ugc 제작
        5.1> "UbiSam.GEM.Configuration.exe" 를 통해서 ugc 를 제작할 때는
            - 필히 배포되는 sample.ugc 를 Open해서 수정한 후 다른이름으로 저장할 것을 권장한다.

        5.2> ugc 파일은 xml 형태의 설정 파일 이므로, vscode 등의 에디터로 열어서 직접 편집할 수도 있으며,
            "UbiSam.GEM.Configuration.exe" 에서 제공되지 않는 기능을 편집하라 수도 있다.
            - ugc 의 기본 구조를 이해해야 AS 에 편리하다.

    6> Host.Simulator 의 활용
        6.1> 많은 고객사들의 문의를 Host.Simulator 로 확인하게 되니,
             왠만한 기능들은 필히 숙지하도록 한다.



2. 유비젬 AS 가이드
    1> Disconncet
        1.1> 최근 설비 제어PC에 보안프로그램이 설치되면서 발생.
            - 키락이 usb 임을 알리고, 보안프로그램 usb 잠금해제 요청 필요하다.

        1.2> usb 1.0 의 경우 usb 가 고장날 수도 있다. 

        1.3> 아주 가끔 제어PC 에 꼽힌 다른 usb 디방이스와 유비젬 키락 1.0 이 충돌을 일으키기도 한다.


    2> 스트림(Stream) , 펑션(Function) 문의 대응
        2.1> SEMI 문서 에서 다음 3문서를 토대로 유비젬이 개발되었으니, 아래 문서를 참고한다.
            - KE005-00-0813.pdf, KE030-00-0611.pdf, KE037-00-0413.PDF

        2.2> SEMI 문서와 번역본 KEXXX-XX-XXXX.pdf 는 라이선스가 걸려있는 문서이므로
            ** [주의] 절대 배포 하지 않는다! **


    3> ugc 문의 대응
        3.1> 고객의 ugc 와 ughsc 파일을 제공가능 한지 문의 하고, 
            제공 받아 "C:\Users\user1\Documents\UbiSam\UbiGEM\Samples\Ubisam.GEM.Sample.CSharp\Ubisam.GEM.Sample.CSharp\bin\Debug" 의
            "UbiSam.GEM.Sample.CSharp.exe" 프로그램을 사용하여 테스트 한다.
            - C:\Program Files\UbiSam\UbiGEM\BIN 폴더의 "UbiGEM.Net.Simulator.Host.exe" 로 테스트를 수행한다.

    4> UbiCOM 문의

            - 대체적으로 키락 문제이다.
                    
        4.2> UbiCOM 의 배포는 기존업체에서 버그문의가 왔을때 
            - "UbiCOM_20250515.zip" 을 배포한다.

            - 저도 아는바가 없어서 대응이 불가능합니다.


    6> _gemDriver.HSMSDriver
        6..1> "C:\Users\user1\Documents\UbiSam\UbiGEM\Samples\Ubisam.GEM.Sample.CSharp"
